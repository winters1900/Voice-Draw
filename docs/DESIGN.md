# 设计文档 · AI 语音绘图工具

> 七牛云暑期赛作品。本文记录：计划支持的指令能力、最终实现情况、未完成部分及原因，以及关键架构与设计决策。

## 一、设计目标

打造一款**纯语音控制**的绘图工具——用户不使用鼠标/键盘，仅凭语音完成绘图创作。围绕赛题三项核心诉求展开设计：

1. **指令理解的准确性与容错性**：口语化、模糊、带填充词/纠错的表达都能正确理解。
2. **语音到绘图的响应延迟**：常用指令需即时响应。
3. **复杂指令的拆解与执行**：一句话包含多个动作时能拆解为有序操作并依次执行。

## 二、整体架构

```
麦克风
  │  音频
  ▼
语音识别(ASR)         七牛 ASR(后端代理) ┐
  │  文本                              ├─ 二选一，可在界面切换
  ▼                   浏览器 Web Speech ┘（无密钥兜底）
指令解析(混合)
  ├─ 本地规则 RuleParser ── 命中 ─→ DrawCommand[]      （零网络延迟）
  └─ 未命中 ─→ 后端 /api/parse ─→ 七牛 LLM ─→ DrawCommand[]（强容错）
  │
  ▼
命令执行 CommandExecutor ──→ CanvasEngine(状态/历史/渲染)
  │  反馈文案
  ▼
语音反馈(TTS)         七牛 TTS(后端代理) / 浏览器 SpeechSynthesis(兜底)
```

**分层与目录**

| 层 | 位置 | 职责 |
| --- | --- | --- |
| 绘图引擎 | `frontend/src/engine` | 图形模型、渲染、选区、撤销/重做、导出 |
| 指令 DSL | `shared/commands.ts` | 前后端共享的结构化指令契约 + 校验 |
| 解析层 | `frontend/src/parser` | 规则快路径 + 混合编排(规则↔LLM) |
| 执行层 | `frontend/src/executor` | DSL → 引擎调用，指代解析、批量排布、反馈文案 |
| 语音层 | `frontend/src/voice` | ASR/TTS 引擎抽象与切换、麦克风控制、字幕 |
| 控制器 | `frontend/src/controller` | 文字/语音统一执行入口 + 日志 + 反馈 |
| 后端 | `backend/src` | 七牛 LLM/ASR/TTS 代理，密钥隔离 |

### 核心抽象：DrawCommand DSL

解析层与执行层之间以结构化指令数组解耦。**一句话可产出多条命令**，这是复杂指令拆解的载体。
规则解析与 LLM 解析产出同一套 DSL，执行层无需关心来源。详见 [`shared/commands.ts`](../shared/commands.ts)。

## 四、关键设计决策

1. **混合解析（延迟 vs 容错的折中）**——高频指令走本地正则，实测 <1ms 命中即时执行；
   仅当规则未命中或低置信度时才请求七牛 LLM（带超时降级）。兼顾“响应延迟”与“理解准确性/容错”。
2. **结构化 DSL 解耦**——规则与 LLM 输出同一套 `DrawCommand`，执行层单一实现；
   LLM 用 `response_format=json_object` + few-shot 强约束，并经 `isValidCommand` 校验过滤，防注入脏数据。
3. **复杂指令拆解**——规则按连接词（然后/接着/，/。）分段；LLM 由 prompt 要求按动作顺序拆成多条命令。
   `count`/`layout` 支持“三个圆排成一行”这类批量+排布语义。
4. **指代消解**——`SelectTarget` 支持 last/all/byType/byColor/byIndex；新建图形即选中，
   使“画个圆…再大一点…变成红色”这类连续指代自然衔接。
5. **健壮的引擎**——快照式撤销/重做，批量创建为单一撤销单元；命令式 API 便于单测（已 37 项单测覆盖）。
6. **降级与兜底**——ASR/TTS 均“七牛优先 + 浏览器原生兜底”；LLM 失败退回规则结果。
   保证**无七牛密钥也能完整演示**绘图链路，有密钥则启用七牛全套能力。

## 五、七牛云服务对接

| 能力 | 接口 | 说明 |
| --- | --- | --- |
| 大模型解析 | `POST {base}/chat/completions`（OpenAI 兼容） | 自然语言 → DrawCommand[] |
| 语音识别 | `POST {base}/voice/asr` | 短指令音频 → 文本（后端代理，密钥隔离） |
| 语音合成 | `POST {base}/voice/tts` | 反馈文案 → 音频 |

`base = https://api.qnaigc.com/v1`，鉴权 `Authorization: Bearer <QINIU_API_KEY>`。密钥仅存后端 `.env`，前端不接触。

## 六、响应延迟设计

- **本地规则快路径**：常用指令（画圆、变色、撤销、清空…）零网络往返，<1ms 解析后立即渲染。
- **LLM 仅用于复杂/模糊指令**，并设超时（默认 9s）降级为 unknown 触发追问，避免长时间无反馈。
- **ASR 流式字幕**：浏览器引擎实时显示中间结果；七牛短指令录音-识别延迟取决于音频时长。
- 日志面板实时展示每条指令的「解析来源 + 耗时」，可直观验证延迟表现。

## 七、测试

- 37 项单元测试（Vitest），覆盖：引擎创建/排布/指代/变换/撤销、规则解析、混合路由、输入归一化、语音控制口令。
- 运行：`npm --prefix frontend run test`。
- 前端 `tsc + vite build`、后端 `tsc` 类型检查均通过。

## 八、未完成部分及原因

| 项 | 原因 |
| --- | --- |
| 自由画笔/手绘曲线 | 语音难以精确描述连续笔迹轨迹，结构化图形性价比更高；列为非目标。 |
| 唤醒词激活 | 连续监听 + 一键开关已满足纯语音演示；唤醒词为锦上添花，优先级较低。 |
| PNG 自动下载落地 | 引擎已具备 `toDataURL` 导出能力，触发自动下载的浏览器交互为后续小项。 |
| 口语线宽调节 | 规则未覆盖“粗一点/细一点”，但 DSL 与 LLM 路径已支持，可由大模型解析。 |
| 七牛流式 ASR（WebSocket） | 流式接口为二进制/gzip 协议且需真实密钥联调；短指令场景改用 REST 更稳，流式作为后续优化。 |

> 联调提示：七牛 ASR/TTS 的精确请求字段以官方文档实测为准，相关 payload 已集中在
> `backend/src/routes/asr.ts` 与 `tts.ts` 各一处，便于按实际协议微调。
